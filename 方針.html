変数・配列・オブジェクト

配列は連想配列を用いる

原則として、配列は何らかのキーを持つこととする。キーに意味を持たせない場合は連番の数字を与える。 

//NGパターン
$arrSome = ['value1', 'value2', 'value3'];
//OKパターン
$arrSome = [
    'key1' => 'value1',
    'key2' => 'value2',
    'key3' => 'value3',
];
$arrSome = [
    1 => 'value1',
    2 => 'value2',
    3 => 'value3',
];

単一のフィールドに複数のデータを保存する場合はjson文字列を用いる

データは連想配列で扱う前提なので、キーを持たない配列を生成することは好ましくない。連想配列をjson文字列に変換したものをフィールドに保存し、データを取得するときに連想配列として取り出せるようにする。フィールド値がjson文字列でもLIKE検索は使える。 

//保存するとき
$fieldData = json_encode($arrSome);
//抽出するとき
$arrSome = json_decode($fieldData, true);

コントローラーのメソッド内で扱うデータの変数名

データが連想配列なら$arrData、オブジェクト型なら$objData、json文字列なら$jsonDataなどとする。SQLで抽出したデータはオブジェクト型だから$objDataに格納する。
複数のモデルを扱う場合は，$objData['user']や$objData['company']など，連想配列にそれぞれのモデルを格納する。

変数名の単数形と複数形

格納するデータが単数・複数に関係なく、変数名は単数形にして保守性を高めるようにしておく。処理の過程で単数形と複数形を分けた方が良い場合に限り複数形を用いる。例えば配列をforeachでループ処理する場合は配列の変数名を複数形にしておいた方が良い場合もあるが、ループ内の変数を$key => $valueと定義して片付くならば、その必要はない。

コンストラクタでモデルを定義する

コントローラーのメソッド内でモデルのインスタンスを行わず、あらかじめコンストラクタでモデルを定義する。モデルに追加の項目が必要になる場合、それぞれのメソッド内で追加すると混乱の原因になるため、コンストラクタに記述する。これにより、モデルはそれぞれのメソッドで単一の構造を持つものとして扱い、コンストラクタを見ればモデルの構造が分かるようにしておく。

最大3階層構成の場合におけるweb.phpの例を以下に示す。XAMPP環境下においてphp artisan serveで動作し，phpUnitテストやduskも動作する。 

<?php

use Illuminate\Support\Facades\Route;
use Illuminate\Support\Str;

/**
  *  -------------------- ルーティングの説明 --------------------
  *  ＊対応は3階層まで
  *  ＊ルーティングにアンダーバーは使わない。ハイフンかつ英数小文字で
  *  0階層   /
  *      コントローラー：IndexController
  *      メソッド：index
  *  1階層   例  /about-us
  *      コントローラー：AboutUsController
  *      メソッド：index
  *  2階層   例  /user/create
  *      コントローラー：UserController
  *      メソッド：create
  *  3階層   例  /dir/company/user-delete
  *      コントローラー：Dir/CompanyController
  *      メソッド：user_delete
  *      3階層構成の場合は第1階層がディレクトリ名となる
  * 
  * 認証が必要なページについて
  * 
  * 例えば認証が必要なページをadminディレクトリ配下に置く場合、admin
  * の1つ下の階層を1階層とする。
  * 
  *  ----------------------------------------------------------
  */

$authDir = 'admin';     //認証が必要なディレクトリ

$branchTunnel = function () use ($authDir) {

    $arrUrl = explode('?', request()->getRequestUri());
    $arrUrl = explode('/', $arrUrl[0]);
    $action = 'index';
    $path = 'App\Http\Controllers\\';

    if (!empty($arrUrl[1]) && $arrUrl[1] === $authDir) {
        array_shift($arrUrl);
        $path = 'App\Http\Controllers\\' . Str::studly($authDir) . '\\';
    }

    switch (true) {
        case empty($arrUrl[1]):
            //0階層構成
            $path .= 'IndexController';
            break;
        case !empty($arrUrl[1]) && empty($arrUrl[2]):
            //1階層構成
            $path .= Str::studly($arrUrl[1]) . 'Controller';
            break;
        case !empty($arrUrl[2]) && empty($arrUrl[3]):
            //2階層構成
            $path .= Str::studly($arrUrl[1]) . 'Controller';
            $action = Str::replace('-', '_', $arrUrl[2]);
            break;
        case !empty($arrUrl[3]) && empty($arrUrl[4]):
            //3階層構成
            $path .= Str::studly($arrUrl[1]) . '\\' . Str::studly($arrUrl[2]) . 'Controller';
            $action = Str::replace('-', '_', $arrUrl[3]);
            break;
        default:
            abort(404);
            break;
    }

    return app()->call("$path@$action");
};

Route::prefix($authDir)->middleware('auth')

        Route::any('/{url}', $branchTunnel)->where('url', '.*');

        return;
    });

    public function step1(Request $request)
    {
        //メソッドを確認してGETとPOST以外は404エラーページに遷移させる
        if (!$request->isMethod('get') && !$request->isMethod('post')) abort(404);

        /*---------- postの場合 ----------*/
        if ($request->isMethod('post')) {

            //json_decode()で連想配列に変換する

            $input = $request->only($this->fillableExt);
            /*---------- 入力値を加工する場合はここに記入 ----------*/
            /*--------------------- ここまで ---------------------*/
            //step1で入力されたデータとstep2/3で入力されたデータをマージしてjson文字列に変換
            $jsonData = json_encode(array_merge($arrData, $input));

            /*---------- 下書き保存 ----------*/
            if ($request->has('action') && $request->action === 'draft') {
                //post→getの遷移はredirect()で行い，with()でセッション値を送る

                return redirect("/$this->dir/$request->transit")->with('jsonData', $jsonData);
            }
        }

        /*---------- getの場合 ----------*/
        //コンストラクタで設定したmodelを格納する
        $objData = $this->model;
        //初期状態のデータをjson文字列に変換

CRUD操作

基本的に以下の4つのメソッドで構成する。
  • index
    • GETのみ受け入れる。検索リクエストもGET。
    • テーブルに保存されたレコードの一覧を表示する。
    • 各レコードをクリックすると詳細画面に遷移する。
    • 新規作成ボタンを設置する。
    • 複数キーワード対応の絞り込み検索窓を設置する。GETメソッドで送信するので@csrfは不要。
  • create
    • GETなら入力フォームの画面を表示し，POSTならレコードを新規保存する。それ以外は棄却する。
    • レコードの新規作成画面を表示する。
    • form要素配下に@csrfが必要。
    • テーブルにレコードを新規作成する。
    • 必要であれば下書きボタンを設置する。
  • detail
    • GETなら詳細画面を表示し，DELETEならレコードを削除する。それ以外は棄却する。
    • レコードのidに基づいて、レコードの詳細を表示する。
    • 編集ボタンを設置し、クリックすると編集画面に遷移する。
    • form要素を設置し、削除ボタンをクリックするとDELETEメソッドで送信する。送信前に確認ダイアログを表示させる。
    • レコードを削除したら、一覧画面に遷移する。
    • form要素配下に@csrfが必要。
    • プレビュー表示が必要な場合はここに設置する。
  • edit
    • GETなら編集フォームの画面を表示し，PATCHならレコードを更新する。それ以外は棄却する。
    • レコードのidに基づいて，編集画面を表示する。
    • form要素配下に@csrfが必要。
    • 更新ボタンをクリックするとPATCHメソッドで送信し，レコードを更新する。
    • 更新が完了したら、一覧画面に遷移する。
更新，削除はPOSTではなく，それぞれPATCH，DELETEメソッドを用いる。POSTで送信するとLaravelのセキュリティ機能が発揮されない。

fillableの扱い

モデルで定義される$fillableはテーブル操作時に書き込みを行うことを許可するカラムを記入する。 

class Company extends Model
{
    use HasFactory;

    protected $fillable = [
        'name',
        'tel',
        'del_flg',
    ];
}
基本的にはテーブルのカラムを全て記入するが，idは含めない。同様にuser_idなど参照元に紐付されたカラムも含めない。こうすることで誤って id を上書きしてしまうミスを防ぐ。参照先のデータを新規作成する場合は以下のようにリレーションメソッドを利用する。 

  $user->company()->create($arrData); //$userはユーザーの情報が入ったオブジェクト
このように書けばLaravel側が自動的にidを書き込んでくれる。
補足であるが，create()やfill()の引数は連想配列なので誤ってオブジェクトを入れないように注意する。オブジェクトのデータを使いたいなら$arrData = $objData->toArray();などとして，いったん連想配列に変換すると良い。
fillableの便利な点は，createの際に引数として与える連想配列にテーブルに関係ない要素が含まれていても、それらを自動的に取り除いてくれるところにある。連想配列は様々な処理のためにもともとテーブルのカラムに無い要素を追加していることも多いが，そういう場合でも気にせず引数としてそのまま放り込んでしまうことができる（動作検証は行うこと）。
ただし、update()の場合はまったく挙動が異なり、fillableに含まれないカラムのデータも更新できる。従ってレコードの更新処理においてupdate()は用いず、fill()とsave()の組み合わせで行う。fill()はfillableを考慮するので、fillableにない項目を誤って書き換えるミスを防ぐことができる。
下のコード例において、$arrDataはuserテーブルとcompanyテーブルのカラムが混合した連想配列であるとする。userとcompanyに同じデータ$arrDataを送っているが、それぞれfill()によってfillableに含まれない項目は自動的に棄却されるため、問題なく更新できる。 

/**
* ユーザー情報更新
* @param array $arrData フォームで入力されたデータ
*/
public function updateUser($arrData): void
{
    // 入力されたデータにパスワードを含む場合はハッシュ化する
    if (!empty($arrData['password'])) {
        $arrData['password'] = bcrypt($arrData['password']);
    }

    // idから該当レコードを抽出
    $objData = $this->find($arrData['id']);
    // fillを用いるとfillableに含まれないカラムが更新されることはない
    $objData->fill($arrData);
        ->save();
    // userモデルが一体一でcompanyモデルを参照するものとする
    $objData->company
        ->fill($arrData);
        ->save();
}

複数キーワードによる検索

以下に複数キーワードによる検索の例を示す。カラムのデータが連想配列のキーの値で保存されている場合、検索で入力される文字列はvalueの値であるため、そのままではLIKE検索できない。そこで、preg_grepとarray_keysを用いることで、valueの値から該当するキーの値を取り出している。 

/**
* ユーザー一覧
* @param string $keyword 絞込検索キーワード（複数）
*/
public function getUserList($keyword): object
{
    // $prefs = ['hokkaido' => '北海道', ・・・, 'okinawa' => '沖縄県']
    $prefs = AddressConst::PREFECTURE;

    // 抽出するカラムの指定は必須。全カラムの指定はuser.*
    $query = $this->select(
        'users.id',
        'users.email',
        'infos.name',
        'infos.pref',
    )
    // joinとleftJoinで結果が同一の場合、明示的にleftJoinを用いる
        ->leftJoin('infos', 'users.id', '=', 'infos.user_id');

    // 検索文字列が存在している場合、検索を行う
    if (!empty($keyword)) {

        // 検索文字列をスペース区切りで配列に分割
        $arrKeyword = $this->splitKeyword($keyword);

        // 配列の各要素を配列で回す、SQLクエリが
        // WHERE (A OR B OR C) AND (A OR B OR C) AND ・・・
        // のような形であれば正解
        foreach ($arrKeyword as $search) {

            // コールバック関数でラップすると、
            // SQLクエリで WHERE ( ・・・ ) に変換される
            $query->where(function ($query) use ($search, $prefs) {

                // 名前とメールアドレスで検索
                $query->where('infos.name', 'LIKE', "%$search%")
                    ->orWhere('users.email', 'LIKE', "%$search%");

                // 都道府県で検索
                //
                // preg_grep()を用いることで
                // $search = '埼玉'または'埼玉県'ならば
                // $prefTemp = ['saitama' => '埼玉県']
                // を取得できる
                // $search = '崎' の場合
                // $prefTemp = ['nagasaki' => '長崎県', 'miyazaki' => '宮崎県']
                // となる。
                $prefTemp = preg_grep('/' . preg_quote($search, '/') . '/u', $prefs);
                // 連想配列からキーを配列として取り出す
                $prefMatches = array_keys($prefTemp);

                // 連想配列をループで回す
                // $search = '崎' なら
                // WHERE (名前 OR メール OR nagasaki OR miyazaki)
                // の形になる
                foreach ($prefMatches as $pref) {
                    $query->orWhere('infos.pref', 'LIKE', "%$pref%");
                }
            });
        }
    }

    $condition = ['users.del_flg' => 0, 'infos.del_flg' => 0];
    $objData = $query->where($condition)->paginate(10);

    return $objData;
}

function setDate(obj) {

    const yearElem = $(`#${obj.yearId}`);
    const monthElem = $(`#${obj.monthId}`);
    const yearMonthElem = $(`#${obj.yearId},#${obj.monthId}`);
    const dayElem = $(`#${obj.dayId}`);
    const inputElem = $(`#${obj.dateId}`);
    const date = inputElem.val() ? new Date(inputElem.val()) : new Date();
    const year = date.getFullYear();
    const month = date.getMonth() + 1;
    const day = date.getDate();
    const endOfMonth = new Date(year, month, 0);

    for (let i = endOfMonth.getFullYear(); i >= 1940; i--) {
        const elem = $(`<option value="${i}">${i}</option>`).appendTo(yearElem);
        if (i === year) elem.prop("selected", true);
    }

    for (let i = 1; i <= 12; i++) {
        const elem = $(`<option value="${i}">${i}</option>`).appendTo(monthElem);
        if (i === month) elem.prop("selected", true);
    }

    for (let i = 1; i <= endOfMonth.getDate(); i++) {
        const elem = $(`<option value="${i}">${i}</option>`).appendTo(dayElem);
        if (i === day) elem.prop("selected", true);
    }

    $(window).on('load', function () {
      inputElem.val(date.toLocaleDateString());
    });

    yearMonthElem.on("change", function () {
        const beginOfMonth = new Date(yearElem.val(), monthElem.val() - 1, 1);
        const endOfMonth = new Date(yearElem.val(), monthElem.val(), 0);
        const year = beginOfMonth.getFullYear();
        const month = beginOfMonth.getMonth() + 1;
        const day = beginOfMonth.getDate();
        dayElem.empty();
        for (let i = 1; i <= endOfMonth.getDate(); i++) {
            dayElem.append(`<option value="${i}">${i}</option>`);
        }
        inputElem.val(`${year}-${month}-${day}`);
    });

    dayElem.on("change", () => {
        const date = `${yearElem.val()}-${monthElem.val()}-${dayElem.val()}`;
        inputElem.val(date);
    });
}

inputやtextareaに「あと〇〇文字」を表示させ，制限文字数に達するとBackSpaceと矢印キー以外の入力を受け付けなくなる。改行文字はカウントしない。

/**
* 残り文字数カウンター
* @param obj {
*      textArea: textareaの要素ID,
*      count: カウンタのP要素ID
*      limit: 最大文字数,
* }
*/
function setCounter(obj) {
    const textElem = $(`#${obj.textArea}`);
    const countElem = $(`#${obj.count}`);
    const limit = obj.limit;
    const regex = /(\r\n|\n|\r)/gm;
    const keys = [
        'Backspace',
        'ArrowUp',
        'ArrowDown',
        'ArrowLeft',
        'ArrowRight',
    ];

    function toggleReadOnly() {
        const text = textElem.val();
        const arrBreak = text.match(regex);
        const countBreak = Array.isArray(arrBreak) ? arrBreak.length : 0;
        if (text.length > limit + countBreak) {
            textElem.val(text.slice(0, limit + countBreak))
                .attr('readonly', true);
            countElem.addClass('edit__alert')
                .text('最大文字数を超えています。これ以上入力できません。');
        } else {
            textElem.removeAttr('readonly');
            const len = limit - text.replace(regex, '').length;
            countElem.removeClass('edit__alert')
                .text(`あと${len}文字`);
        }
    }

    $(window).on('load', toggleReadOnly);
    textElem.on('input', toggleReadOnly)
        .on('keydown', function (e) {
            if (textElem.attr('readonly') && keys.includes(e.key)) {
                toggleReadOnly();
            }
        });
}

function toggleDisabled(obj) {
    const selectElem = $(`#${obj.select}`);
    const condition = obj.condition;
    const targetElem = $(`#${obj.target}`);

    function toggleAttr() {
        if (condition.includes(selectElem.val())) {

        }
    }

    getSendCoordinate(obj, image) {

        let result = {};
        const scaleX = obj.send.maxWidth / image.width;     //倍率
        const scaleY = obj.send.maxHeight / image.height;

        switch (obj.mode) {
            case "nocrop":
                //試しに幅で合わせてみたら画像高さがはみでる場合
                if (image.height * scaleX >= obj.send.maxHeight) {
                    //高さで合わせる
                    return {
                        sx: 0,
                        sy: 0,
                        sw: image.width,
                        sh: image.height,
                        dx: Math.abs(obj.send.maxWidth - image.width * scaleY) / 2,
                        dy: 0,
                        dw: image.width * scaleY,
                        dh: image.height * scaleY,
                    };
                } else {
                    //画像幅がはみでる場合幅で合わせる
                    return {
                        sx: 0,
                        sy: 0,
                        sw: image.width,
                        sh: image.height,
                        dx: 0,
                        dy: Math.abs(obj.send.maxHeight - image.height * scaleX) / 2,
                        dw: image.width * scaleX,
                        dh: image.height * scaleX,
                    };
                }
                break;
            case "crop":

                if (image.height * scaleX >= obj.send.maxHeight) {
                    //幅で合わせて画像上下をカット
                    const trimY = Math.abs(obj.send.maxHeight / scaleX - image.height);
                    return {

Laravelのインストール

インストール

以下，PowerShellより操作。 

composer create-project laravel/laravel example-app
config\app.phpを以下に修正。 

'timezone' => 'Asia/Tokyo',
'locale' => 'ja',
'faker_locale' => 'ja_JP',
XAMPP環境で動かす場合は以下でサーバーを起動する。 

php artisan serve
日本語化についてはLaravel Langを参照。 

composer require laravel-lang/common --dev
php artisan lang:add ja
php artisan lang:update


// Storageのシンボリックリンク

php artisan storage:link

// Breezeのインストール

composer require laravel/breeze --dev

// fakerのインストール

composer require fakerphp/faker
