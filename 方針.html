変数・配列・オブジェクト

配列はすべて連想配列とする

特別な事情がない限り、配列は何らかのキーを持つことで統一する。キーに特別な意味を持たせない場合は、番号を振っておく。 

//NGパターン
$arrSome = ['value1', 'value2', 'value3'];
//OKパターン
$arrSome = [
    'key1' => 'value1',
    'key2' => 'value2',
    'key3' => 'value3',
];
$arrSome = [
    1 => 'value1',
    2 => 'value2',
    3 => 'value3',
];

単一のフィールドに複数のデータを保存する場合はjson文字列を用いる

データが単純な場合はスラッシュ区切りでも良いけど、データは連想配列で扱う前提なので、キーを持たない配列を生成することは好ましくない。json文字列に統一しておいて、データを出し入れするときに連想配列として扱えるようにする。json文字列の状態でもLIKE検索は使える。 

//保存するとき
$fieldData = json_encode($arrSome);
//抽出するとき
$arrSome = json_decode($fieldData, true);

コントローラーのメソッド内で扱うデータの変数名

データが連想配列なら$arrData、オブジェクト型なら$objData、json文字列なら$jsonDataとする。SQLで抽出したデータはオブジェクト型だから$objDataに格納する。
複数のモデルを扱う場合は，$objData['user']や$objData['company']など，連想配列にそれぞれのモデルを格納する。

コンストラクタでモデルを定義する

コントローラーにおいて，基本的にメソッド内でモデルのインスタンスを行わなず，コンストラクタでモデルを定義する。モデルにあらかじめ用意されている項目だけでは足りずに追加することも多いが，個々のメソッド内でそれをやると混乱の元になるのでコンストラクタに記述し，クラス全体で共通するものとして設定しておく。 

use App\Models\Company;

private $model = null;
private $fillableExt = [];  //fillableの項目を拡張したもの，fillableExtendedの略

public function __construct()
{
    $this->model = new User;
    $this->fillableExt = array_merge($this->model->getFillable(), [
        /*----------- 項目を追加する場合はここに記入 -----------*/
        'status',
        'role',
        /*--------------------- ここまで ---------------------*/
    ]);
    $this->model->setAttrs(array_fill_keys($this->fillableExt, null));

    /*---------- 初期値を与える場合はここに記入 ----------*/
    $this->model->status = 'private';
    $this->model->role = 1;
    /*-------------------- ここまで --------------------*/
}
モデルをインスタンスしただけでは項目自体存在しない状態なので、getFillable()でモデル内に記述されている$fillableを取得し項目を追加した上で、setAttrs()ですべての項目にいったんnullを設定している。 

//たとえばモデルで以下のように書かれているとき
protected $fillable = [
    'email',
    'password',
];
//setAttrsを実行
$this->model = [
    'email' => null,
    'password' => null,
    'status' => null,
    'role' => null,
];
項目を追加したり、nullではなく初期値を設定したい場合は指定の場所に記述する。コントローラーのメソッド内では$this->modelを呼び出してデータの読み書きをしたり、$this->model->メソッド名()の形でモデルに操作を加えたりする。
このようにして、モデルのデータはオブジェクトとして扱うことで統一しておく。後述するが、データにページ情報を付与して取り出した場合、そのデータをtoArray()などで連想配列に変換するとページ情報が消えてしまうのが理由である。コントローラーのメソッドによってデータをオブジェクトとしたり連想配列としたりすると混乱の元になる。モデルに関係するデータは、コンストラクタからモデルを引っ張ってきて、そこに集約しておく。
また、オブジェクトであればデータに何らかの処理を加えたい場合にメソッドを実行すれば良いので、利便性が高い。 

$objData = $this->model;      //モデルをいったん$objDataに格納する（可読性を確保）
$role = $objData->role;       //データを読み込む場合

$items = $this->fillableExt;  //拡張された項目の配列を取得する場合
$items = $objData->getFillable();   //モデルの$fillableを取得する場合

/*---------- メソッドの実行 ----------*/
// 以下の例ならモデル側に
// public function newEntry() { ～ }
// を記述する

$objData->newEntry();
ちなみにsetAttrs()は独自のメソッドでモデル内に設置するが、後述するtraitを利用することでコード自体は別ファイルに記述し、モデル同士で共通するメソッドとしてコードを再利用する。 

/**
* モデルのプロパティを連想配列でまとめて格納する
*/
public function setAttrs($attrs): void
{
    foreach ($attrs as $key => $value) {
        $this->$key = $value;
    }
}

ルーティングについて

name()は使わない。従って、ヘルパーのroute()も使わない。ルート相対パスで直接指定する。 アドレスと各コントローラー・メソッドの詳細はweb.php内に書いてある。注意が必要なのは 

Route::any('/{url}', function () {
の部分で、コントローラー内の一つのメソッドに対し、GET、POST、PATCH、DELETEなどいずれでもアクセスできるようになっている。コントローラーを記述するときに、不要なものについてはメソッドの初めで拒否し、以降の処理に進まないようにする。 

public function confirm(Request $request)
{
    if ($request->isMethod('get') &&
        !$request->session()->has('jsonData')) abort(404);
    if (!$request->isMethod('get')
        && !$request->isMethod('post')) abort(404);

    /*  以降の処理  */
}
上の例ではGETとPOSTを受け入れ、GETの場合セッション値を必須としている。条件を満たさない場合はabort(404)で404エラーページを表示する。
web.phpは書き方がやや教科書的でない部分があるが，この書き方ならXAMPP環境下においてphp artisan serveで動作するし，phpUnitテストやduskが動く。

例えばcreate→confirm→completeの流れがあるとき、create(GET) → create(POST) → confirm(GET) → confirm(POST) → complete(GET)という流れで進行していく。POSTからGETに遷移するときはview()ではなくredirect()を使う。同時にwith()を用いてセッション値を渡しておく。
view()でもデータを送って画面を表示することはできるが、セッションを介してデータを送る方がセキュリティ上好ましい。画面の表示と画面の遷移は明確に区別し、表示はview()、遷移はredirect()とする。

<button type="submit" name="transit" class="editSubmit" value="step2">次に進む</button>

<input type="hidden" name="jsonData" value="{{ old('jsonData', $jsonData) }}">
ポイントになるのは、上の部分。button要素にnameとvalueを設置することで、種々の指令を送る。
何らかの処理を要求する場合はnameをactionとしてvalueに指示内容を記述する。コントローラー側には$request->actionの値がdraftとして送られるので、if文で判定して処理を書く。当然ながらvalueに異なる値を設定したbutton要素を追加すれば、様々な指令を送ることができるようになる。
画面遷移の指令はnameをtransitとしてvalueに遷移先のURLを指定する。こちらもbuttonを追加して複数の遷移先を設置することができる。

    public function step1(Request $request)
    {
        //メソッドを確認してGETとPOST以外は404エラーページに遷移させる
        if (!$request->isMethod('get') && !$request->isMethod('post')) abort(404);

        /*---------- postの場合 ----------*/
        if ($request->isMethod('post')) {
            //jsonDataにはstep1～step3の全項目のデータが入っている
            //json_decode()で連想配列に変換する
            $arrTemp = json_decode($request->jsonData, true);
            $input = $request->only($this->fillableExt);
            /*---------- 入力値を加工する場合はここに記入 ----------*/
            /*--------------------- ここまで ---------------------*/
            //step1で入力されたデータとstep2/3で入力されたデータをマージしてjson文字列に変換
            $jsonData = json_encode(array_merge($arrData, $input));

            /*---------- 下書き保存 ----------*/
            if ($request->has('action') && $request->action === 'draft') {
                //post→getの遷移はredirect()で行い，with()でセッション値を送る
                return redirect("/$this->dir/draft-complete")->with('jsonData', $jsonData);
            }

            /*---------- ページ遷移 ----------*/
            if ($request->has('transit')) {
                //transitには遷移先のurlが格納されている。こちらも同様にwith()でセッション値を送る
                return redirect("/$this->dir/$request->transit")->with('jsonData', $jsonData);
            }
        }

        /*---------- getの場合 ----------*/
        //コンストラクタで設定したmodelを格納する
        $objData = $this->model;
        //初期状態のデータをjson文字列に変換
        $jsonData = json_encode($objData);

        /*---------- セッションが存在する場合セッション値を反映 ----------*/
        //step2で「前に戻る」をクリックして遷移してきた場合はセッション値を持っている
        //それ以外の画面から遷移してきた場合はセッション値は存在しない
        if ($request->session()->has('jsonData')) {
            //セッション値を取得：pull()で取得と削除を同時に行う
            //このデータはビュー側の隠しフィールドに格納する。そして，
            //ビュー側でsubmitした時に，上の「postの場合」の箇所で再度
            //受け取ることになる
            $jsonData = $request->session()->pull('jsonData');
            //json文字列を連想配列に変換して，オブジェクトに書き込む
            //このデータはビューで表示するために利用する
            $objData->setAttrs(json_decode($jsonData, true));
        }

        //この時点で，セッションが存在しなければ$objDataは初期状態であり，
        //セッションが存在していればセッション値の内容を反映したデータになっている

        /*---------- 表示用データを加工する場合はここ ----------*/
        /*----------------- ここまで -------------------------*/

        return view("$this->dir.step1", [
            'objData' => $objData,
            'jsonData' => $jsonData,
        ]);

          /*--------------------- ここまで ---------------------*/
          $jsonData = json_encode(array_merge($arrData, $input));

          /*---------- 下書き保存 ----------*/
          if ($request->has('action') && $request->action === 'draft') {
              return redirect("/$this->dir/draft-complete")->with('jsonData', $jsonData);
          }

          /*---------- ページ遷移 ----------*/
          if ($request->has('transit')) {
              return redirect("/$this->dir/$request->transit")->with('jsonData', $jsonData);
          }
      }

      /*---------- getの場合 ----------*/
      $objData = $this->model;
      $jsonData = json_encode($objData);
      
      /*---------- セッションが存在する場合セッション値を反映 ----------*/
      if ($request->session()->has('jsonData')) {
          $jsonData = $request->session()->pull('jsonData');
          $objData->setAttrs(json_decode($jsonData, true));
      }

      /*---------- 表示用データを加工する場合はここ ----------*/
      /*----------------- ここまで -------------------------*/

      return view("$this->dir.confirm", [
            'objData' => $objData,
            'jsonData' => $jsonData,
        ]);
    }

    public function complete(Request $request)
    {
        if (!$request->isMethod('get')) abort(404);
        //completeの場合はセッション値を必須とする
        if ($request->isMethod('get') && !$request->session()->has('jsonData')) abort(404);

        /*---------- getの場合 ----------*/
        $jsonData = $request->session()->pull('jsonData');
        //ここでデータをテーブルに保存する
        //保存するメソッドはモデル側に記述する
        $objData = $this->model;
        $objData->newEntry($jsonData, 'public');

        return view("$this->dir.complete");
    }

    {
        if (!$request->isMethod('get')) abort(404);

管理画面のCRUD操作

基本的に以下の4つのメソッドで構成する。
  • index
    • テーブルに保存されたレコードの一覧を表示する。
    • 各レコードをクリックすると詳細画面に遷移する。
    • 新規作成ボタンを設置する。
    • 複数キーワード対応の絞り込み検索窓を設置する。
  • create　レコードの新規作成画面を表示する。POSTメソッドでDBにレコードを新規保存する。
  • detail　レコードのidに基づいて、レコードの詳細を表示する。
    • 編集ボタンを設置する。editに遷移。
    • 削除ボタンを設置する。form要素を設置してDELETEメソッドで送信する。submitの前に確認ダイアログを表示させる。
    • プレビュー表示が必要な場合はここに設置する。
  • edit　レコードのidに基づいて，編集画面を表示する。PATCHメソッドで送信し，レコードを更新する。
更新，削除についてはPOSTではなく，それぞれPATCH，DELETEメソッドを使う。使わないとLaravelのセキュリティ機能が発揮されない。

indexメソッドはGETのみ受け入れる（検索リクエストもGET）。
createメソッドはGETなら入力フォームの画面を表示し，POSTならレコードを新規保存する。それ以外は受け付けない。
detailメソッドはGETなら詳細画面を表示し，DELETEならレコードを削除する。それ以外は受け付けない。パラメーターとしてidが必要。
editメソッドはGETなら編集フォームの画面を表示し，PATCHならレコードを更新する。それ以外は受け付けない。パラメーターとしてidが必要。

fillableの扱いについて

モデルで定義される$fillableはテーブル操作時に書き込みを行うことを許可するカラムを記入する。 

class Company extends Model
{
    use HasFactory;

    protected $fillable = [
        'name',
        'tel',
        'del_flg',
    ];
}
基本的にはテーブルのカラムを全て記入するが，idは含めない。同様にuser_idなど参照元に紐付されたカラムも含めない。こうすることで誤って id を上書きしてしまうミスを防ぐ。参照先のデータを新規作成する場合は以下のようにリレーションメソッドを利用する。 

  $user->company()->create($arrData); //$userはユーザーの情報が入ったオブジェクト
このように書けばLaravel側が自動的にidを書き込んでくれる。
補足であるが，create()やfill()の引数は連想配列なので誤ってオブジェクトを入れないように注意する。オブジェクトのデータを使いたいなら$arrData = $objData->toArray();などとして，いったん連想配列に変換すると良い。
fillableの便利な点は，createの際に引数として与える連想配列にテーブルに関係ない要素が含まれていても、それらを自動的に取り除いてくれるところにある。連想配列は様々な処理のためにもともとテーブルのカラムに無い要素を追加していることも多いが，そういう場合でも気にせず引数としてそのまま放り込んでしまうことができる（動作検証は行うこと）。
ただし、update()の場合はまったく挙動が異なり、fillableに含まれないカラムのデータも更新できる。従ってレコードの更新処理においてupdate()は用いず、fill()とsave()の組み合わせで行う。fill()はfillableを考慮するので、fillableにない項目を誤って書き換えるミスを防ぐことができる。
下のコード例において、$arrDataはuserテーブルとcompanyテーブルのカラムが混合した連想配列であるとする。userとcompanyに同じデータ$arrDataを送っているが、それぞれfill()によってfillableに含まれない項目は自動的に除外されるため、問題なく更新できる。 

/**
* ユーザー情報更新
* @param array $arrData フォームで入力されたデータ
*/
public function updateUser($arrData): void
{
    // 入力されたデータにパスワードを含む場合はハッシュ化する
    if (!empty($arrData['password'])) {
        $arrData['password'] = bcrypt($arrData['password']);
    }

    // idから該当レコードを抽出
    $objData = $this->find($arrData['id']);
    // fillを用いるとfillableに含まれないカラムが更新されることはない
    $objData->fill($arrData);
        ->save();
    // userモデルが一体一でcompanyモデルを参照するものとする
    $objData->company
        ->fill($arrData);
        ->save();
}

namespace App\Traits;

trait ModelTrait {

    }

パスワードの扱いについて

Breezeの使用を前提とする。テーブルに保存するパスワードは他のデータと扱いが異なる。まず、パスワードはモデル内において$hiddenに記述する。こうするとで、SQL処理でパスワードを取り出すことができなくなる。パスワードはログイン認証時のみ必要なデータであり、それ以外の場面でパスワードを取り出して利用してはいけない。 Userテーブル等からレコードを取り出して編集する場合、パスワード入力欄は空のままで良いし、入力欄を非表示にしてボタンをクリックしたときにだけ入力できるようにすることが望ましい（後述のパーツ集を参照）。 

protected $fillable = [
  'password',
];

protected $hidden = [
  'password',
];
一方で、$fillableにもパスワードを記述し、新規登録と更新で用いる（取り出せないが書き込むことはできる）。パスワードを書き込む際にはbcrypt($request->password)として、ハッシュ化した上で書き込む。ハッシュ化しないでテーブルに書き込むとログインできなくなるので注意する。

バリデーションの書き方

バリデーションはフォームリクエストバリデーションを用いる。詳しくは公式を参照。
authorize()はreturn trueとしておかないとエラーになるので注意が必要。
リクエストのデータの前処理を行いたい場合はprepareForValidation()を設置する。加工したデータはmerge()で他のデータと統合する。必要なければ関数自体設置しない。

namespace App\Http\Requests;

class UserRequest extends FormRequest
{

    {

    {
        $name = $this->request->name;
        $this->merge([
            'slug' => Str::slug($this->slug),
        ]);

    public function rules(): array
    {
        if (!$this->isMethod('post')) return [];

        return [
            'name' => 'required|string|max:80',
            'someColumn' => 'exclude',
        ];
    }
  
    public function attributes(): array
    {
        return [
            'name' => '名前',
        ];
    }

        ];
    }
}

/**
* セレクトボックスで日付を選択
* 連結された日付を入力するinput要素をhiddenで設置する
* @param obj object{
*      yearId: string,
*      monthId: string,
*      dayId: string,
*      dateId: string}　年月日と日付のid
*/
function setDate(obj) {

    const yearElem = $(`#${obj.yearId}`);
    const monthElem = $(`#${obj.monthId}`);
    const yearMonthElem = $(`#${obj.yearId},#${obj.monthId}`);
    const dayElem = $(`#${obj.dayId}`);
    const inputElem = $(`#${obj.dateId}`);
    const date = inputElem.val() ? new Date(inputElem.val()) : new Date();
    const year = date.getFullYear();
    const month = date.getMonth() + 1;
    const day = date.getDate();
    const endOfMonth = new Date(year, month, 0);

    for (let i = endOfMonth.getFullYear(); i >= 1940; i--) {
        const elem = $(`<option value="${i}">${i}</option>`).appendTo(yearElem);
        if (i === year) elem.prop("selected", true);
    }

    for (let i = 1; i <= 12; i++) {
        const elem = $(`<option value="${i}">${i}</option>`).appendTo(monthElem);
        if (i === month) elem.prop("selected", true);
    }

    for (let i = 1; i <= endOfMonth.getDate(); i++) {
        const elem = $(`<option value="${i}">${i}</option>`).appendTo(dayElem);
        if (i === day) elem.prop("selected", true);
    }

    $(window).on('load', function () {
      inputElem.val(date.toLocaleDateString());
    });

    yearMonthElem.on("change", function () {
        const beginOfMonth = new Date(yearElem.val(), monthElem.val() - 1, 1);
        const endOfMonth = new Date(yearElem.val(), monthElem.val(), 0);
        const year = beginOfMonth.getFullYear();
        const month = beginOfMonth.getMonth() + 1;
        const day = beginOfMonth.getDate();
        dayElem.empty();
        for (let i = 1; i <= endOfMonth.getDate(); i++) {
            dayElem.append(`<option value="${i}">${i}</option>`);
        }
        inputElem.val(`${year}-${month}-${day}`);
    });

    dayElem.on("change", () => {
        const date = `${yearElem.val()}-${monthElem.val()}-${dayElem.val()}`;
        inputElem.val(date);
    });
}

/**
* 残り文字数カウンター
* @param obj obj{textArea: string, count: string} textareaとカウンタを表示する要素のID
*/
function setCounter(obj) {
    const maxLength = 200;  //最大文字数
    const textElem = $(`#${obj.textArea}`);
    const countElem = $(`#${obj.count}`);
    $(window).on('load', function () {
        countElem.text(`あと${maxLength}文字`);
    });

    textElem.on('input', function () {
        const text = textElem.val().replace(/(\r\n|\n|\r)/gm, "");
        if (text.length >= maxLength) {
            textElem.attr('readonly', true);
        } else {
            textElem.removeAttr('readonly');
        }
        const len = maxLength - text.length;
        countElem.text(`あと${len}文字`);
    });

    textElem.on('keydown', function (e) {
        if (textElem.attr('readonly')) {
            if (e.keyCode === 8 || e.keyCode === 46) {
                textElem.removeAttr('readonly');
            }
        }
    });
}

Laravelのセットアップ

インストール

以下，PowerShellより操作。 

composer create-project laravel/laravel example-app

ローカル開発サーバーの起動

xampp環境で動かす場合は以下。 

php artisan serve

日本語化

日本語化についてはLaravel Langを参照。 

composer require laravel-lang/common --dev

php artisan lang:add ja

php artisan lang:update

Storageのシンボリックリンク


php artisan storage:link

Breezeのインストール


composer require laravel/breeze --dev
