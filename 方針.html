変数・配列・オブジェクト

配列はすべて連想配列とする

特別な事情がない限り、配列は何らかのキーを持つことで統一する。キーに特別な意味を持たせない場合は、番号を振っておく。 

//NGパターン
$arrSome = ['value1', 'value2', 'value3'];
//OKパターン
$arrSome = [
    'key1' => 'value1',
    'key2' => 'value2',
    'key3' => 'value3',
];
$arrSome = [
    1 => 'value1',
    2 => 'value2',
    3 => 'value3',
];

単一のフィールドに配列を保存する場合はjson文字列を用いる

データが単純な場合はスラッシュ区切りでも良いけど、json文字列に統一しておいた方が連想配列をそのまま扱えるし、複雑なデータ形式でも対応できる。 

//保存するとき
$fieldData = json_encode($arrSome);
//抽出するとき
$arrSome = json_decode($fieldData, true);

コントローラーのメソッド内で扱うデータの変数名

データが連想配列なら $arrData、オブジェクト型なら $objData、json文字列なら $jsonData とする。SQLで抽出したデータはオブジェクト型だから $objData に格納する。基本的には $objData を使う。

コントローラー内のコンストラクタについて

コントローラー内のコンストラクタは以下のようになっている。 

private $model = null;
private $fillableExt = [];  //メソッド間の受け渡し項目

public function __construct()
{
    $model = new User();
    $this->fillableExt = array_merge($model->getFillable(), [
        /*  メソッド間の受け渡し項目を追加する場合はここに記入  */
        'role',
        'tel',
        /*  ここまで  */]);
    //全項目にnullを代入
    $model->setAttrs(array_fill_keys($this->fillableExt, null));
    /*  初期値を与える場合はここに記入  */
    $model->role = 1;
    /*  ここまで  */
}
モデルをインスタンスしただけでは項目自体存在しない状態なので、getFillable()でモデル内に記述されている$fillableを取得し、setAttrs()ですべての項目にnullを設定している。 

//たとえば
protected $fillable = [
    'email',
    'password',
];
//なら
$this->model = [
    'email' => null,
    'password' => null,
];
項目を追加したり、nullではなく初期値を設定したい場合は指定の場所に記述する。$this->modelをオブジェクトの初期状態としてメソッド内で利用する。 また、原則的にモデルに関係するデータはオブジェクト型で統一する（連想配列とオブジェクト型は要素の指定方法が異なるので、統一しておいてあとで考えないで済むようにしておく）。 ちなみにsetAttrs()は独自のメソッドでモデル内に記述している。 

/**
* モデルのプロパティを連想配列でまとめて格納する
*/
public function setAttrs($attrs): void
{
    foreach ($attrs as $key => $value) {
        $this->$key = $value;
    }
}

ルーティング

ルーティングについて

->name()は使わない。従って、アクセスするときもroute()は使わない。ルート相対パスで直指定する。 アドレスと各コントローラーの詳細はweb.php内に書いてある。注意が必要なのは 

Route::any('/{url}', function () {
の部分で、コントローラー内の一つのメソッドに対し、get、post、patch、deleteなどいずれでもアクセスできるようになっている。 従って意図しないものについてはメソッドの初めに弾いて、後の処理に進まないようにする必要がある。 

public function confirm(Request $request)
{
    if ($request->isMethod('get') &&
        !$request->session()->has('jsonData')) abort(404);
    if (!$request->isMethod('get')
        && !$request->isMethod('post'))  abort(404);

    /*  以降の処理  */
}
上の例ではgetとpostを受け入れ、getの場合セッション値を必須としている。条件を満たさない場合はabort(404)で404エラーページを表示するようにしている。

cssとJavaScriptの分離

bladeの中にcssやJavaScriptのコードが入ると保守性が下がるので、原則として別ファイルに記述して、layoutにファイルを読み込むように追記しておく。bootstrapやjQueryを使いたいときも同様。 JavsScriptのコードは必ず関数でラップして、意図しないページでコードが読み込まれないようにしておく。
jsファイル 

function myFunc() {
  /*  何らかの処理  */
}
bladeファイル 

・・・・
</x-guest-layout>
<script>
  myFunc();  //blade側から関数を呼び出す
</script>

JavaScriptに変数を渡す

コントローラーからJavaScriptに変数を渡すときの書き方は以下。
・変数が1つの場合
bladeファイル 

<script>
  myFunc(<?php echo $varSome; ?>);
</script>
・変数が複数の場合
bladeファイル 

<script>
myFunc({
  'key1': <?php echo $arrSome; ?>,
  'key2': <?php echo $varSome; ?>
});
</script>
変数が複数の場合はオブジェクトで渡すと楽。JavaScriptの方では
jsファイル 

function myFunc(obj) {
  const name = obj.name;
  const tel = obj.tel;

  /*  何らかの処理  */
}
のように書いて値を取り出す。 注意点として、JavaScriptに渡したいデータが連想配列やオブジェクトの場合、json_encode()でjson文字列化してbladeに送ること。 

return view('user.create', [
  'objData' => $objData,
  'jsonSome' => json_encode($arrSome),
]);

bladeのバリデーションエラー表示は@errorを使う


@error('email') <p class="alert">{{ $message }}</p> @enderror
